# edikt:invariant:compile

Generate or regenerate directive sentinel blocks (`[edikt:directives:start/end]: #`) for Invariant Records. Sentinels are what `/edikt:gov:compile` reads — without them, compile falls back to extraction, which is slower and less accurate.

See [ADR-009](../../docs/architecture/decisions/ADR-009-invariant-record-terminology.md) for the Invariant Record template contract. Invariant Records are edikt's formal artifact type for the governance artifact documenting a hard architectural constraint.

## Compile Schema (ADR-008)

This command writes sentinel blocks that conform to the **three-list schema with hash-based caching** locked in by [ADR-008](../../docs/architecture/decisions/ADR-008-deterministic-compile-and-three-list-schema.md). The block has three directive lists and three metadata fields:

- **`directives:`** — auto-generated by this command from the invariant's Statement/Rule section. Rewritten on every body change.
- **`manual_directives:`** — user-authored rules. NEVER touched by this command. Preserved across all regenerations.
- **`suppressed_directives:`** — auto directives the user rejected. NEVER touched by this command. Filtered out at `/edikt:gov:compile` time.
- **`source_hash`** — SHA-256 hex of the invariant body with the directives block excluded, after normalization (CRLF→LF, trailing whitespace stripped per line).
- **`directives_hash`** — SHA-256 hex of the canonical `directives:` list items joined with `\n`.
- **`compiler_version`** — edikt semver at compile time (e.g., `"0.3.0"`).

**Critical behavior rules (enforced by ADR-008):**
1. This command NEVER reads `manual_directives:` or `suppressed_directives:`.
2. This command NEVER filters `directives:` based on `suppressed_directives:`. Filtering happens at `gov:compile` time.
3. On a cache hit (both hashes match), this command does NOTHING — no Claude call, no file writes.
4. On a hand-edit detection (`source_hash` matches, `directives_hash` does not), this command runs the interactive interview in Section 3c. In headless mode it fails with error code 2 unless `--strategy=regenerate` or `--strategy=preserve` was passed.

The full formal schema lives at `docs/internal/product/prds/PRD-001-spec/schema.yaml`. The hash algorithm reference is at `docs/internal/product/prds/PRD-001-spec/hash-reference.md`.

## Arguments

- `$ARGUMENTS` — optional invariant ID (e.g., `INV-002`) plus optional strategy flags:
  - No argument — process all active invariants.
  - `INV-NNN` — process a single invariant.
  - `--regenerate` — force fresh Claude regeneration regardless of hash match. Discards hand-edits in `directives:`. Preserves `manual_directives:` and `suppressed_directives:`.
  - `--strategy=regenerate` — headless override. Discard hand-edits, rewrite from body.
  - `--strategy=preserve` — headless override. Skip this invariant entirely.

## Instructions

### 0. Config Guard

If `.edikt/config.yaml` does not exist, output:
```
No edikt config found. Run /edikt:init to set up this project.
```
And stop.

### 1. Resolve Paths

Read `.edikt/config.yaml`. Resolve:
- Invariants: `paths.invariants` (default: `docs/architecture/invariants`)
- edikt version: read `~/.edikt/VERSION`. Use this for `compiler_version:` in any block written by this run. Fall back to `edikt_version:` in `.edikt/config.yaml` if `~/.edikt/VERSION` is missing.

### 2. Determine Scope

**With `$ARGUMENTS` containing an `INV-NNN` token** — locate the invariant file matching the given ID. Search for a file whose name contains the ID prefix. If not found:
```
Invariant not found: {id}
Run: ls {invariants_path}/*.md to see available invariants.
```

**Without an INV ID** — glob all `*.md` files in `{invariants_path}`. Filter to those with `status: active` (check frontmatter or `**Status:** Active` in the body). Include files with no `status:` field for backwards compatibility.

Skip any file with `status: retired`, `status: superseded`, or `status: proposed`. Per ADR-009, only `Active` invariants compile into governance.

If no active invariants found:
```
No active invariants found in {invariants_path}.
```

**Parse strategy flags** from `$ARGUMENTS`:
- If `--regenerate` is present, set `force_regenerate=true`.
- If `--strategy=regenerate` is present, set `strategy=regenerate`.
- If `--strategy=preserve` is present, set `strategy=preserve`.
- If both `--strategy=` variants are present, error:
  ```
  Cannot combine --strategy=regenerate with --strategy=preserve.
  ```
  And stop.
- If neither is present, `strategy=interactive` (default).

**Detect headless mode**: if `EDIKT_HEADLESS=1` is set or no interactive TTY is available, mark `headless=true`. In headless mode, `strategy=interactive` becomes an error when the interview path would otherwise fire.

### 3. Process Each Invariant

For each invariant in scope, execute the algorithm below exactly. The order of steps matters.

#### 3a. Compute current hashes

Read the invariant file content as UTF-8.

**Compute `current_source_hash`** — SHA-256 of the body with the directives block excluded:

1. Remove the entire `[edikt:directives:start]: #` … `[edikt:directives:end]: #` block, inclusive of both sentinel lines. If the block is not present, "body" is the whole file.
2. Replace all `\r\n` with `\n`. Then replace all remaining `\r` with `\n`.
3. For each line, strip trailing whitespace matching `[ \t\v\f]+$`. Do NOT strip leading whitespace.
4. Compute SHA-256 of the UTF-8 bytes of the result. Encode as lowercase hex, 64 characters.

Reference bash:
```bash
awk '
  /^\[edikt:directives:start\]/ { skip=1; next }
  /^\[edikt:directives:end\]/   { skip=0; next }
  !skip { print }
' "{file}" | tr -d '\r' | sed 's/[[:space:]]*$//' | shasum -a 256 | awk '{print $1}'
```

**Compute `current_directives_hash`** — SHA-256 of the canonical auto `directives:` list:

1. Parse the YAML content between the sentinel lines.
2. Extract the `directives:` list only. Do NOT include `manual_directives:` or `suppressed_directives:`.
3. Join list items with `\n` (LF), in document order.
4. Compute SHA-256 of the UTF-8 bytes. Encode as lowercase hex.

#### 3b. Read stored hashes and classify

Parse the existing directives block (if any) and extract:
- `stored_source_hash`, `stored_directives_hash`, `stored_compiler_version` — from the new schema
- `content_hash` — legacy v0.2.x field

Classify the invariant into one of these states:

| State | Condition | Next action |
|---|---|---|
| **Fresh** | No sentinel block, OR block has no `source_hash` AND no `content_hash` | Section 3d (Generate) |
| **Legacy v0.2.x** | Block has `content_hash:` but none of `source_hash`/`directives_hash`/`compiler_version` | Silent migration: Section 3d, treated as first compile under new schema |
| **Clean** | Both hashes match | FAST PATH: skip. No Claude call, no writes. |
| **Body-changed** | `stored_source_hash != current_source_hash` | SLOW PATH: Section 3d. Update both hashes on completion. |
| **Hand-edited** | `stored_source_hash == current_source_hash` AND `stored_directives_hash != current_directives_hash` | Section 3c (Interview). |
| **Forced** | User passed `--regenerate` or `--strategy=regenerate` | SLOW PATH regardless of hashes. |

**Compiler version drift**: if `stored_compiler_version` is not null and differs from the current edikt version, print:
```
ℹ INV-005 was compiled by edikt {stored_compiler_version} (current: {current}). Run with --regenerate to refresh if needed.
```
Do NOT auto-regenerate. User opts in explicitly.

#### 3c. Hand-edit interview (Hand-edited state only)

The user has modified the `directives:` list after the last compile wrote it.

1. Regenerate the auto directives from the body (Section 3d).
2. Diff the regenerated list against the current `directives:` list.
3. For each line in the current list that is NOT in the regenerated list: a **hand-added line**.
4. For each line in the regenerated list that is NOT in the current list: a **hand-deleted auto directive**.

**If `headless=true` and `strategy=interactive`** — fail loud:
```
❌ /edikt:invariant:compile {INV-ID}

Cannot run in headless mode: hand-edited lines detected in directives:
that require user intervention.

  Lines in directives: but not in fresh auto output:
    + "{hand-added line}"
    ...

  Lines in fresh auto output but not in directives::
    - "{hand-deleted line}"
    ...

Options:
  1. Run interactively: /edikt:invariant:compile {INV-ID}
  2. Override (destructive):  --strategy=regenerate
       → Discards hand-edits, rewrites directives: from body
  3. Override (preserve):     --strategy=preserve
       → Leaves file unchanged, skips this invariant

Exiting with error (2).
```
Exit this invariant with error 2.

**If `strategy=regenerate`** — discard hand-edits. Treat as SLOW PATH. Report: `{file}: hand-edits discarded, regenerated from body`.

**If `strategy=preserve`** — skip this invariant entirely. Report: `{file}: hand-edits preserved, regeneration deferred`.

**If `strategy=interactive` (and `headless=false`)** — run the interview.

For each hand-added line in `directives:`, ask:

```
🔍 /edikt:invariant:compile {INV-ID}

Reviewing directives: against current invariant body...
Found {n} lines that don't match auto-generated output.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
({i}/{n}) Hand-added line in directives::

    "{hand-added line text}"

This line is in directives: but wouldn't be generated from the
current invariant body. What should I do?

  [1] Move to manual_directives: (keep as user-authored rule)
  [2] Add to suppressed_directives: (pick this only if compile
      had generated this before and you want to prevent
      regeneration)
  [3] Delete entirely
  [4] Edit invariant body first so this becomes an auto directive
  [5] Skip for now (leave in directives: — will be overwritten)

Choice [1]:
```

For each hand-deleted auto directive, ask:

```
({i}/{n}) Deleted auto directive:

    "{deleted line text}"

Compile still wants to regenerate this from the body. What
should I do?

  [1] Add to suppressed_directives: (keep deleted — compile
      filters it out on every run via gov:compile)
  [2] Let compile regenerate (deletion was accidental)
  [3] Edit invariant body first to remove the source

Choice [1]:
```

After the interview, write the resolved state (Section 3e).

#### 3d. Generate auto directives from the invariant body

Read the `## Statement` section (per ADR-009 template) or the `## Rule` section (v0.2.x legacy format). Invariant Records per ADR-009 use `## Statement` for the constraint and `## Rationale` for the reasoning. The Statement section contains the directive-shaped content.

Rules for generating directives:
1. Invariant statements already use declarative, absolute language per the ADR-009 writing guide — preserve it exactly. Do NOT add hedging.
2. If the Statement is a single sentence, emit one directive derived from it.
3. If the Statement has multiple clauses, emit one directive per clause.
4. Enforcement mechanisms from the `## Enforcement` section may become additional directives (e.g., "Use `authz.Check()` for all authorization decisions" becomes a directive).
5. Do NOT include rationale prose from `## Rationale` as directives.
6. Each directive ends with `(ref: {INV-ID})`.
7. If the legacy `## Rule` section is used (v0.2.x), it already uses MUST/NEVER language — preserve it exactly.
8. **"No exceptions." reinforcement:** If the invariant's `## Statement` section uses absolute quantifiers — "every", "all", "total", "no ... path", "no ... exception", "never" — append "No exceptions." to the generated directive text, before the `(ref:)` suffix. Example: `"Every data access MUST be tenant-scoped. No exceptions. (ref: INV-012)"`. This phrase acts as absolute reinforcement that prevents Claude from rationalizing edge cases. Only apply to invariant directives — ADRs and guidelines may legitimately have exceptions.
9. **Reminder generation:** For each generated directive, identify the ACTION the constraint governs (writing SQL, creating files, returning errors, logging, starting goroutines, etc.) and emit a reminder line in the format: `"Before {action} → {check} (ref: {INV-ID})"`. Store these in a `reminders:` list inside the sentinel block, alongside `directives:`. Example: `"Before writing SQL → MUST include tenant_id in WHERE clause (ref: INV-012)"`. Cap at 2 reminders per invariant.
10. **Verification item generation:** For each generated directive that can be expressed as a grep command or file inspection, emit a checklist item in the format: `"[ ] {what to check} (ref: {INV-ID})"`. Store in a `verification:` list inside the sentinel block. Only include items that are mechanically verifiable — skip directives that require reading logic. Cap at 3 items per invariant.

Derive `paths:` from any path globs declared in the invariant's metadata or from the `## Implementation` section if it names specific file patterns. Default to `"**/*"` for universal constraints.

Derive `scope:` — invariants apply to all activities by default per ADR-009:
```yaml
scope:
  - planning
  - design
  - review
  - implementation
```
Because invariants are non-negotiable across the entire SDLC. If the invariant is narrower (e.g., implementation-only), override this default based on the invariant's `## Implementation` or `## Enforcement` sections.

**Validate cross-references:** for every generated directive that references an ADR, INV, or guideline, confirm that identifier exists in the source document. Strip any fabricated references.

#### 3e. Write the sentinel block

Assemble the new block content:

```yaml
[edikt:directives:start]: #
source_hash: {current_source_hash}
directives_hash: {sha256 of the new canonical directives list}
compiler_version: "{edikt version from Section 1}"
paths:
  - {glob patterns}
scope:
  - planning
  - design
  - review
  - implementation
directives:
  - {directive} (ref: {INV-ID})
  - {directive} (ref: {INV-ID})
reminders:
  - "Before {action} → {check} (ref: {INV-ID})"
verification:
  - "[ ] {what to check} (ref: {INV-ID})"
manual_directives:
  {preserved verbatim from existing block if present; otherwise empty list []}
suppressed_directives:
  {preserved verbatim from existing block if present; otherwise empty list []}
[edikt:directives:end]: #
```

**Backward compatibility rules:**
- If the existing block has `content_hash:` (legacy) and nothing else new, silently drop it and write the new schema.
- Preserve `manual_directives:` and `suppressed_directives:` byte-for-byte if present.
- If neither exists, write empty YAML lists (`[]`).

**Computing `directives_hash` for the new block**: take the newly-written `directives:` list, canonicalize via `join('\n')`, compute SHA-256.

**Insert or replace** the block in the invariant file, inside the `## Directives` section if present, or append before the closing `---` line. Write atomically.

### 5. Implementation notes for Claude (the executing agent)

You are reading these instructions and MUST execute them literally:

- **Never write to `manual_directives:` or `suppressed_directives:`**. These lists belong to the user.
- **Never filter `directives:` based on `suppressed_directives:`**. That is `gov:compile`'s job.
- **Never skip the hash computation**.
- **Never silently auto-resolve hand-edits**. Interview or fail loud.
- **Always compute hashes deterministically** per Section 3a.
- **Always update both hashes** when writing a new block.
- **Always preserve `compiler_version:` drift behavior**: warn, don't auto-regenerate.

---

REMEMBER: Invariant Records compile into the Non-Negotiable Constraints section of the governance index — they appear at the top and bottom of every governance file to exploit primacy and recency bias. The `## Statement` section is the source of truth — preserve declarative language exactly. Invariants scope to all activities by default because they cannot be violated under any circumstances. This command is the ONLY writer of the `directives:`, `reminders:`, and `verification:` lists. `manual_directives:` and `suppressed_directives:` belong to the user. Append "No exceptions." to directives derived from absolute-language Statements. Contract locked in ADR-008 and ADR-009.
