# edikt:adr:compile

Generate or regenerate directive sentinel blocks (`[edikt:directives:start/end]: #`) for ADRs. Sentinels are what `/edikt:gov:compile` reads — without them, compile falls back to extraction, which is slower and less accurate.

## Compile Schema (ADR-008)

This command writes sentinel blocks that conform to the **three-list schema with hash-based caching** locked in by [ADR-008](../../docs/architecture/decisions/ADR-008-deterministic-compile-and-three-list-schema.md). The block has three directive lists and three metadata fields:

- **`directives:`** — auto-generated by this command from the ADR body. Rewritten on every body change. Hand-edits are detected via `directives_hash` and resolved via the interactive interview flow.
- **`manual_directives:`** — user-authored rules. NEVER touched by this command. Preserved across all regenerations. Used for rules compile missed or couldn't infer.
- **`suppressed_directives:`** — auto directives the user rejected. NEVER touched by this command. Filtered out at `/edikt:gov:compile` time, not here.
- **`source_hash`** — SHA-256 hex of the ADR body with the directives block excluded, after normalization (CRLF→LF, trailing whitespace stripped per line). Tells this command whether the body has changed since last compile.
- **`directives_hash`** — SHA-256 hex of the canonical `directives:` list items joined with `\n`. Tells this command whether the user has hand-edited the auto list.
- **`compiler_version`** — edikt semver at compile time (e.g., `"0.3.0"`). Used to detect algorithm drift across edikt upgrades.

**Critical behavior rules (enforced by ADR-008):**
1. This command NEVER reads `manual_directives:` or `suppressed_directives:`. They are invisible to the hash comparison and to the generation logic. Only the user writes and reads them; only `gov:compile` consumes them.
2. This command NEVER filters `directives:` based on `suppressed_directives:`. Filtering happens at `gov:compile` time.
3. On a cache hit (both hashes match), this command does NOTHING — no Claude call, no file writes.
4. On a hand-edit detection (`source_hash` matches, `directives_hash` does not), this command runs the interactive interview in Section 3c. In headless mode it fails with error code 2 unless `--strategy=regenerate` or `--strategy=preserve` was passed.

The full formal schema lives at `docs/internal/product/prds/PRD-001-spec/schema.yaml`. The hash algorithm reference implementation is at `docs/internal/product/prds/PRD-001-spec/hash-reference.md`.

## Arguments

- `$ARGUMENTS` — optional ADR ID (e.g., `ADR-003`) plus optional strategy flags:
  - No argument — process all accepted ADRs.
  - `ADR-NNN` — process a single ADR.
  - `--regenerate` — force fresh Claude regeneration regardless of hash match. Discards hand-edits in `directives:`. Preserves `manual_directives:` and `suppressed_directives:`. Use after an edikt upgrade that materially changes compile logic.
  - `--strategy=regenerate` — headless override. When a hand-edit is detected, discard the user's edits and write fresh auto directives. Implies `--regenerate` behavior for the affected ADR only. Does NOT bypass the body hash check — if body is unchanged and directives are user-edited, this strategy still regenerates them.
  - `--strategy=preserve` — headless override. When a hand-edit is detected, skip this ADR entirely (no changes written, exit 0 for that ADR). Used in CI contexts where you want to defer the decision to an interactive session.

## Instructions

### 0. Config Guard

If `.edikt/config.yaml` does not exist, output:
```
No edikt config found. Run /edikt:init to set up this project.
```
And stop.

### 1. Resolve Paths

Read `.edikt/config.yaml`. Resolve:
- Decisions: `paths.decisions` (default: `docs/architecture/decisions`)
- edikt version: read `~/.edikt/VERSION` (installed edikt version). Use this value for `compiler_version:` in any block written by this run. If `~/.edikt/VERSION` is missing, fall back to the `edikt_version:` value in `.edikt/config.yaml`.

### 2. Determine Scope

**With `$ARGUMENTS` containing an `ADR-NNN` token** — locate the ADR file matching the given ID. Search for a file whose name contains the ID prefix. If not found:
```
ADR not found: {id}
Run: ls {decisions_path}/*.md to see available ADRs.
```

**Without an ADR ID** — glob all `*.md` files in `{decisions_path}`. Filter to those with `status: accepted` (check frontmatter `status:` field, or fall back to `**Status:** accepted` in the body).

If no accepted ADRs found:
```
No accepted ADRs found in {decisions_path}.
```

**Parse strategy flags** from `$ARGUMENTS`:
- If `--regenerate` is present, set `force_regenerate=true`.
- If `--strategy=regenerate` is present, set `strategy=regenerate`.
- If `--strategy=preserve` is present, set `strategy=preserve`.
- If both are present, error:
  ```
  Cannot combine --strategy=regenerate with --strategy=preserve.
  ```
  And stop.
- If neither is present, `strategy=interactive` (default).

**Detect headless mode**: if the environment variable `EDIKT_HEADLESS=1` is set, OR if the interactive TTY is not available, mark `headless=true`. In headless mode, `strategy=interactive` becomes an error when the interview path would otherwise fire (see Section 3c).

### 3. Process Each ADR

For each ADR in scope, execute the algorithm below exactly. The order of steps matters — the hash comparison gates the decision tree.

#### 3a. Compute current hashes

Read the ADR file content as UTF-8.

**Compute `current_source_hash`** — SHA-256 of the body with the directives block excluded:

1. Remove the entire `[edikt:directives:start]: #` … `[edikt:directives:end]: #` block, inclusive of both sentinel lines. If the block is not present, the "body" is the whole file.
2. Replace all `\r\n` with `\n`. Then replace all remaining `\r` with `\n`.
3. For each line, strip trailing whitespace matching `[ \t\v\f]+$`. Do NOT strip leading whitespace.
4. Compute SHA-256 of the UTF-8 bytes of the result. Encode as lowercase hex, 64 characters.

Reference bash implementation:
```bash
awk '
  /^\[edikt:directives:start\]/ { skip=1; next }
  /^\[edikt:directives:end\]/   { skip=0; next }
  !skip { print }
' "{file}" | tr -d '\r' | sed 's/[[:space:]]*$//' | shasum -a 256 | awk '{print $1}'
```

**Compute `current_directives_hash`** — SHA-256 of the canonical auto `directives:` list:

1. Parse the YAML content between the sentinel lines.
2. Extract the `directives:` list. If the block is not present or the list is absent/empty, the canonical representation is the empty string.
3. Join list items with `\n` (LF), in document order. Do NOT include `manual_directives:` or `suppressed_directives:` in this computation.
4. Compute SHA-256 of the UTF-8 bytes of the joined string. Encode as lowercase hex.

#### 3b. Read stored hashes and classify

Parse the existing directives block (if any) and extract:
- `stored_source_hash` — the `source_hash:` field, or `null` if missing
- `stored_directives_hash` — the `directives_hash:` field, or `null` if missing
- `stored_compiler_version` — the `compiler_version:` field, or `null` if missing
- `content_hash` — legacy v0.2.x field, or `null` if missing

Classify the ADR into one of these states:

| State | Condition | Next action |
|---|---|---|
| **Fresh** | No sentinel block at all, OR sentinel block has no `source_hash` AND no `content_hash` | First-compile path: go to Section 3d (Generate) |
| **Legacy v0.2.x** | Sentinel has `content_hash:` but none of `source_hash`/`directives_hash`/`compiler_version` | Silent migration: go to Section 3d (Generate), treat as first compile under the new schema |
| **Clean** | `stored_source_hash == current_source_hash` AND `stored_directives_hash == current_directives_hash` | FAST PATH: skip. No Claude call, no writes. Mark as "up to date" in the report. |
| **Body-changed** | `stored_source_hash != current_source_hash` | SLOW PATH: go to Section 3d (Generate). On completion, update both hashes. |
| **Hand-edited** | `stored_source_hash == current_source_hash` AND `stored_directives_hash != current_directives_hash` | Go to Section 3c (Interview). |
| **Forced** | User passed `--regenerate` or `--strategy=regenerate` | Treat as SLOW PATH regardless of hashes. |

**Compiler version drift**: if `stored_compiler_version` is not null and differs from the current edikt version, print a one-line info message:
```
ℹ ADR-007 was compiled by edikt {stored_compiler_version} (current: {current}). Run with --regenerate to refresh if needed.
```
Do NOT auto-regenerate on version drift. The user opts in explicitly.

#### 3c. Hand-edit interview (Hand-edited state only)

The user has modified the `directives:` list after the last compile wrote it. Determine what changed by diffing the stored directives against the current directives:

1. Parse `stored_directives_hash` — but since we only stored the hash, not the list itself, we cannot diff directly against the stored list. Instead, **regenerate the auto directives from the body (as in Section 3d) and diff the regenerated list against the current `directives:` list in the file**.
2. For each line in the current `directives:` list that is NOT in the regenerated list: this is a **hand-added line**.
3. For each line in the regenerated list that is NOT in the current `directives:` list: this is a **hand-deleted auto directive**.

**If `headless=true` and `strategy=interactive`** — fail loud:
```
❌ /edikt:adr:compile {ADR-ID}

Cannot run in headless mode: hand-edited lines detected in directives:
that require user intervention.

  Lines in directives: but not in fresh auto output:
    + "{hand-added line}"
    ...

  Lines in fresh auto output but not in directives::
    - "{hand-deleted line}"
    ...

Options:
  1. Run interactively: /edikt:adr:compile {ADR-ID}
  2. Override (destructive):  --strategy=regenerate
       → Discards hand-edits, rewrites directives: from body
  3. Override (preserve):     --strategy=preserve
       → Leaves file unchanged, skips this ADR

Exiting with error (2).
```
Exit this ADR with error 2 (continue processing other ADRs if batch mode).

**If `strategy=regenerate`** — discard the hand-edits. Treat as SLOW PATH. Go to Section 3d with the regenerated list. Report: `{file}: hand-edits discarded, regenerated from body`.

**If `strategy=preserve`** — skip this ADR entirely. Leave the file unchanged. Report: `{file}: hand-edits preserved, regeneration deferred`.

**If `strategy=interactive` (and `headless=false`)** — run the interview, one question per detected change.

For each hand-added line in `directives:`, ask:

```
🔍 /edikt:adr:compile {ADR-ID}

Reviewing directives: against current ADR body...
Found {n} lines that don't match auto-generated output.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
({i}/{n}) Hand-added line in directives::

    "{hand-added line text}"

This line is in directives: but wouldn't be generated from the
current ADR body. What should I do?

  [1] Move to manual_directives: (keep as user-authored rule)
  [2] Add to suppressed_directives: (pick this only if compile
      had generated this before and you want to prevent
      regeneration)
  [3] Delete entirely (remove from the file)
  [4] Edit ADR body first so this becomes an auto directive
  [5] Skip for now (leave in directives: — will be overwritten
      next compile)

Choice [1]:
```

Accept user choice. Apply the action to the planned new state of the file (do not write yet).

For each hand-deleted auto directive, ask:

```
({i}/{n}) Deleted auto directive:

    "{deleted line text}"

Compile still wants to regenerate this from the body. What
should I do?

  [1] Add to suppressed_directives: (keep deleted — compile
      filters it out on every run via gov:compile)
  [2] Let compile regenerate (deletion was accidental)
  [3] Edit ADR body first to remove the source of this directive

Choice [1]:
```

After the interview, write the resolved state (Section 3e).

#### 3d. Generate auto directives from the ADR body

Read the `## Decision` section of the ADR. Extract every enforceable statement — statements that prescribe or prohibit a specific behavior, tool, pattern, or structure. Transform each into a directive:

Rules for generating directives:
1. Hard constraints use MUST or NEVER (uppercase) with a one-clause reason.
2. Name specific things — namespaces, tools, patterns, file paths, thresholds.
3. One directive per line.
4. Each directive ends with `(ref: {ADR-ID})`.
5. Drop rationale prose, context, and alternatives — directives only.
6. **Reminder generation:** For each directive, identify the ACTION the decision governs (creating files, choosing patterns, importing packages, etc.) and emit a reminder: `"Before {action} → {check} (ref: {ADR-ID})"`. Store in a `reminders:` list inside the sentinel block. Cap at 2 reminders per ADR.
7. **Verification item generation:** For each directive that can be verified by grep or file inspection, emit a checklist item: `"[ ] {what to check} (ref: {ADR-ID})"`. Store in a `verification:` list. Cap at 3 items per ADR. Skip directives that require reading logic.

Derive `paths:` by scanning the project directory for file types and locations relevant to the decision's topic.

Derive `scope:` from the decision domain:
- Cross-cutting architecture decisions: `[planning, design, review]`
- Implementation patterns: `[implementation, review]`
- Tooling or infrastructure: `[implementation]`

**Validate cross-references:** for every generated directive that references another ADR or INV, confirm that identifier exists in the source document. Strip any fabricated references — keep the directive text if it's otherwise accurate.

#### 3e. Write the sentinel block

Assemble the new block content:

```yaml
[edikt:directives:start]: #
source_hash: {current_source_hash}
directives_hash: {sha256 of the new canonical directives list}
compiler_version: "{edikt version from Section 1}"
paths:
  - {glob patterns}
scope:
  - {activity scopes}
directives:
  - {directive} (ref: {ADR-ID})
  - {directive} (ref: {ADR-ID})
reminders:
  - "Before {action} → {check} (ref: {ADR-ID})"
verification:
  - "[ ] {what to check} (ref: {ADR-ID})"
manual_directives:
  {preserved verbatim from the existing block if present; otherwise empty list []}
suppressed_directives:
  {preserved verbatim from the existing block if present; otherwise empty list []}
[edikt:directives:end]: #
```

**Backward compatibility rules:**
- If the existing block has `content_hash:` (legacy v0.2.x) and nothing else new, silently drop `content_hash:` and write the new schema. No user warning is needed — this is the expected migration path.
- If the existing block has `manual_directives:` or `suppressed_directives:`, preserve them byte-for-byte (including comments and formatting inside the list items). This command MUST NOT touch them.
- If the existing block has neither manual nor suppressed lists, write both as empty YAML lists (`manual_directives: []`, `suppressed_directives: []`) so the schema is complete.

**Computing `directives_hash` for the new block**: take the newly-written `directives:` list, canonicalize via `join('\n')`, compute SHA-256. Use this value for the `directives_hash:` field in the same write.

**Insert or replace** the block in the ADR file, inside the `## Directives` section if present, or append before the closing `---` line. Write atomically: read the full file, compute the new content, write it in a single operation.

### 5. Implementation notes for Claude (the executing agent)

You are reading these instructions and MUST execute them literally. Do not shortcut or approximate. In particular:

- **Never write to `manual_directives:` or `suppressed_directives:`**. These lists belong to the user. If the existing block has them, preserve byte-for-byte. If the existing block lacks them, write empty lists (`[]`).
- **Never filter `directives:` based on `suppressed_directives:`**. Filtering is `gov:compile`'s job. You write the full raw auto output.
- **Never skip the hash computation** just because "the file looks unchanged". The hash is the source of truth for whether to regenerate.
- **Never silently auto-resolve hand-edits**. Hand-edited state MUST either run the interview (interactive mode) or fail loud with the `--strategy=` hint (headless mode).
- **Always compute hashes deterministically** per Section 3a. Normalization steps matter. A single spurious space in the wrong place produces a different hash and triggers a false regeneration.
- **Always update both hashes** when you write a new block. Writing `directives:` without updating `directives_hash:` leaves the file in an inconsistent state that will trigger a spurious interview on the next compile.
- **Always preserve `compiler_version:` drift behavior**: warn, don't auto-regenerate. Version drift is informational.

---

REMEMBER: This command is the ONLY writer of the `directives:` list. `manual_directives:` and `suppressed_directives:` belong to the user — never read, never write, never filter against them. Fast path (hash match) means no Claude call. Interview path means no silent auto-resolution. Headless mode means fail loud with `--strategy=` hints. This contract is locked in ADR-008.
