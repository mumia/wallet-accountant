# edikt:guideline:compile

Generate or regenerate directive sentinel blocks (`[edikt:directives:start/end]: #`) for team guidelines. Sentinels are what `/edikt:gov:compile` reads — without them, compile falls back to extraction, which is slower and less accurate.

This command brings guidelines to parity with ADRs and Invariant Records in v0.3.0. Guidelines become first-class governance sources: their `## Rules` sections (which use MUST/NEVER language per the `/edikt:guideline:new` template contract) compile into directives that `/edikt:gov:compile` reads alongside ADR and invariant directives.

## Compile Schema (ADR-008)

This command writes sentinel blocks that conform to the **three-list schema with hash-based caching** locked in by [ADR-008](../../docs/architecture/decisions/ADR-008-deterministic-compile-and-three-list-schema.md). The block has three directive lists and three metadata fields:

- **`directives:`** — auto-generated by this command from the guideline's `## Rules` section. Rewritten on every body change.
- **`manual_directives:`** — user-authored rules. NEVER touched by this command. Preserved across all regenerations.
- **`suppressed_directives:`** — auto directives the user rejected. NEVER touched by this command. Filtered out at `/edikt:gov:compile` time.
- **`source_hash`** — SHA-256 hex of the guideline body with the directives block excluded, after normalization.
- **`directives_hash`** — SHA-256 hex of the canonical `directives:` list items joined with `\n`.
- **`compiler_version`** — edikt semver at compile time.

**Critical behavior rules (enforced by ADR-008):**
1. This command NEVER reads `manual_directives:` or `suppressed_directives:`.
2. This command NEVER filters `directives:` based on `suppressed_directives:`. Filtering happens at `gov:compile` time.
3. On a cache hit (both hashes match), this command does NOTHING — no Claude call, no file writes.
4. On a hand-edit detection, this command runs the interactive interview in Section 3c. In headless mode it fails with error 2 unless `--strategy=regenerate` or `--strategy=preserve` was passed.

The full formal schema lives at `docs/internal/product/prds/PRD-001-spec/schema.yaml`. The hash algorithm reference is at `docs/internal/product/prds/PRD-001-spec/hash-reference.md`.

## Arguments

- `$ARGUMENTS` — optional guideline slug (e.g., `error-handling`, `api-design`) plus optional strategy flags:
  - No argument — process all guidelines in the configured directory.
  - `{slug}` — process a single guideline.
  - `--regenerate` — force fresh Claude regeneration regardless of hash match.
  - `--strategy=regenerate` — headless override: discard hand-edits, rewrite from body.
  - `--strategy=preserve` — headless override: skip this guideline entirely.

## Instructions

### 0. Config Guard

If `.edikt/config.yaml` does not exist, output:
```
No edikt config found. Run /edikt:init to set up this project.
```
And stop.

### 1. Resolve Paths

Read `.edikt/config.yaml`. Resolve:
- Guidelines: `paths.guidelines` (default: `docs/guidelines`)
- edikt version: read `~/.edikt/VERSION`. Use this for `compiler_version:`. Fall back to `edikt_version:` in config if missing.

If the guidelines directory does not exist:
```
No guidelines directory found at {guidelines_path}.
Run /edikt:guideline:new to create your first guideline.
```
And stop.

### 2. Determine Scope

**With `$ARGUMENTS` containing a slug** — locate the guideline file matching `{slug}.md` or a file whose name contains the slug as a prefix. If not found:
```
Guideline not found: {slug}
Run: ls {guidelines_path}/*.md to see available guidelines.
```

**Without a slug** — glob all `*.md` files in `{guidelines_path}`. Process every file that has a `## Rules` section.

Skip any file explicitly marked as draft or superseded (check for `**Status:** draft` or `**Status:** superseded` in the frontmatter or near the top).

If no guidelines found:
```
No guidelines found in {guidelines_path}.
Run /edikt:guideline:new to create a guideline.
```

**Parse strategy flags** from `$ARGUMENTS`:
- `--regenerate` → `force_regenerate=true`
- `--strategy=regenerate` → `strategy=regenerate`
- `--strategy=preserve` → `strategy=preserve`
- Both strategy variants → error and stop.
- Neither → `strategy=interactive` (default).

**Detect headless mode** (`EDIKT_HEADLESS=1` or no TTY) and set `headless=true` if applicable.

### 3. Process Each Guideline

For each guideline in scope, execute the algorithm exactly.

#### 3a. Compute current hashes

Read the guideline file as UTF-8.

**Compute `current_source_hash`** — SHA-256 of the body with the directives block excluded:

1. Remove the entire `[edikt:directives:start]: #` … `[edikt:directives:end]: #` block, inclusive of both sentinels.
2. Replace `\r\n` with `\n`, then remaining `\r` with `\n`.
3. Strip trailing whitespace per line.
4. SHA-256 of UTF-8 bytes, lowercase hex.

Reference bash:
```bash
awk '
  /^\[edikt:directives:start\]/ { skip=1; next }
  /^\[edikt:directives:end\]/   { skip=0; next }
  !skip { print }
' "{file}" | tr -d '\r' | sed 's/[[:space:]]*$//' | shasum -a 256 | awk '{print $1}'
```

**Compute `current_directives_hash`** — SHA-256 of the canonical auto `directives:` list:

1. Parse the YAML between sentinels.
2. Extract ONLY the `directives:` list. Not `manual_directives:`, not `suppressed_directives:`.
3. Join items with `\n` in document order.
4. SHA-256 of UTF-8 bytes, lowercase hex.

#### 3b. Read stored hashes and classify

Parse the existing directives block (if any) and extract `stored_source_hash`, `stored_directives_hash`, `stored_compiler_version`, and `content_hash` (legacy).

Classify:

| State | Condition | Next action |
|---|---|---|
| **Fresh** | No block, or no source_hash and no content_hash | Section 3d (Generate) |
| **Legacy v0.2.x** | Has `content_hash:` but none of the new fields | Silent migration: Section 3d |
| **Clean** | Both hashes match | FAST PATH: skip |
| **Body-changed** | `source_hash` mismatch | SLOW PATH: Section 3d |
| **Hand-edited** | `source_hash` matches, `directives_hash` doesn't | Section 3c (Interview) |
| **Forced** | User passed `--regenerate` or `--strategy=regenerate` | SLOW PATH |

**Compiler version drift**: if stored version differs from current, print one-line info message. Do NOT auto-regenerate.

#### 3c. Hand-edit interview

1. Regenerate the auto directives from the body (Section 3d).
2. Diff regenerated vs current `directives:`.
3. Identify hand-added lines and hand-deleted auto directives.

**If `headless=true` and `strategy=interactive`** — fail loud:
```
❌ /edikt:guideline:compile {slug}

Cannot run in headless mode: hand-edited lines detected in directives:
that require user intervention.

  Lines in directives: but not in fresh auto output:
    + "{hand-added line}"
    ...

  Lines in fresh auto output but not in directives::
    - "{hand-deleted line}"
    ...

Options:
  1. Run interactively: /edikt:guideline:compile {slug}
  2. Override (destructive):  --strategy=regenerate
  3. Override (preserve):     --strategy=preserve

Exiting with error (2).
```
Exit this guideline with error 2.

**If `strategy=regenerate`** — discard hand-edits, treat as SLOW PATH.
**If `strategy=preserve`** — skip this guideline entirely.

**If `strategy=interactive` and `headless=false`** — run the interview.

For each hand-added line:
```
🔍 /edikt:guideline:compile {slug}

Reviewing directives: against current guideline body...
Found {n} lines that don't match auto-generated output.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
({i}/{n}) Hand-added line in directives::

    "{hand-added line text}"

This line is in directives: but wouldn't be generated from the
current guideline body. What should I do?

  [1] Move to manual_directives: (keep as user-authored rule)
  [2] Add to suppressed_directives: (only if compile had
      generated this before and you want to prevent regeneration)
  [3] Delete entirely
  [4] Edit guideline body first so this becomes an auto directive
  [5] Skip for now (leave in directives: — will be overwritten)

Choice [1]:
```

For each hand-deleted auto directive:
```
({i}/{n}) Deleted auto directive:

    "{deleted line text}"

Compile still wants to regenerate this from the body. What
should I do?

  [1] Add to suppressed_directives: (keep deleted)
  [2] Let compile regenerate (deletion was accidental)
  [3] Edit guideline body first to remove the source

Choice [1]:
```

After the interview, write the resolved state (Section 3e).

#### 3d. Generate auto directives from the guideline body

Read the `## Rules` section. Guidelines are already directive-shaped — every rule in that section uses MUST or NEVER language per the `/edikt:guideline:new` template contract. Translate each bullet into a directive:

Rules for generating directives:
1. The `## Rules` section uses MUST or NEVER language — preserve it exactly.
2. One directive per bullet in the Rules list.
3. Each directive ends with `(ref: {guideline-slug})`.
4. If a bullet uses soft language ("should", "prefer", "try to", "consider", "aim to"), do NOT generate a directive for it. Warn:
   ```
   ⚠ Skipped soft rule in {file}: "{bullet text}"
     Guidelines should use MUST/NEVER. Either rewrite the rule or omit it.
   ```
5. Ignore bullets in other sections (`## Examples`, `## Rationale`, `## When NOT to apply`) — only `## Rules` contributes directives.
6. **Reminder generation:** For each directive, identify the ACTION the guideline governs and emit a reminder: `"Before {action} → {check} (ref: {guideline-slug})"`. Store in a `reminders:` list inside the sentinel block. Cap at 2 reminders per guideline.
7. **Verification item generation:** For each directive that can be verified by grep or file inspection, emit a checklist item: `"[ ] {what to check} (ref: {guideline-slug})"`. Store in a `verification:` list. Cap at 3 items per guideline.

Derive `paths:` from the guideline's topic slug using this lookup table, falling back to `"**/*"` if unrecognized:

| Topic keyword | Default paths glob |
|---|---|
| `error-handling`, `errors`, `exceptions` | `**/*.{go,ts,tsx,js,jsx,py,rb,php,rs,java,kt}` |
| `testing`, `tests` | `**/*_test.{go,ts,tsx,js,jsx,py}`, `**/test_*.py`, `**/*.test.{ts,tsx,js,jsx}` |
| `logging`, `observability` | `**/*.{go,ts,tsx,js,jsx,py,rb,php,rs,java,kt}` |
| `api-design`, `api`, `endpoints` | `**/handlers/**`, `**/routes/**`, `**/controllers/**`, `**/*.{go,ts,tsx,js,jsx,py}` |
| `database`, `sql`, `migrations` | `**/*.sql`, `**/migrations/**`, `**/repository/**` |
| `frontend`, `ui`, `components` | `**/*.{tsx,jsx,vue,svelte,css,scss}`, `**/components/**` |
| `security`, `auth` | `**/auth/**`, `**/*.{go,ts,tsx,js,jsx,py}` |
| `performance`, `perf` | `**/*.{go,ts,tsx,js,jsx,py,rb,php,rs,java,kt}` |
| (none of the above) | `**/*` |

Derive `scope:` from the topic:

| Topic keyword | Default scope |
|---|---|
| `testing`, `tests` | `[testing, review]` |
| `api-design`, `api` | `[design, implementation, review]` |
| `security`, `auth` | `[design, implementation, review, security]` |
| `performance`, `perf` | `[design, implementation, review, performance]` |
| (all others) | `[implementation, review]` |

**Validate cross-references:** for every generated directive that references an ADR, INV, or another guideline, confirm that identifier exists. Strip fabricated references.

#### 3e. Write the sentinel block

Assemble the new block:

```yaml
[edikt:directives:start]: #
source_hash: {current_source_hash}
directives_hash: {sha256 of the new canonical directives list}
compiler_version: "{edikt version from Section 1}"
paths:
  - {glob patterns}
scope:
  - {scope tags}
directives:
  - {directive} (ref: {guideline-slug})
  - {directive} (ref: {guideline-slug})
reminders:
  - "Before {action} → {check} (ref: {guideline-slug})"
verification:
  - "[ ] {what to check} (ref: {guideline-slug})"
manual_directives:
  {preserved verbatim from existing block if present; otherwise empty list []}
suppressed_directives:
  {preserved verbatim from existing block if present; otherwise empty list []}
[edikt:directives:end]: #
```

**Backward compatibility rules:**
- If the existing block has `content_hash:` (legacy) only, drop it silently and write the new schema.
- If the existing block has no `reminders:` or `verification:` lists (pre-v0.3.0 blocks), write both as empty lists (`[]`). These lists are populated on the next `<artifact>:compile` run.
- Preserve `manual_directives:` and `suppressed_directives:` byte-for-byte if present.
- If neither exists, write empty YAML lists (`[]`).

**Computing `directives_hash`**: take the newly-written `directives:` list, join with `\n`, SHA-256.

**Insert or replace** the block at the end of the guideline file before any footer (e.g., `*Created by edikt:guideline — ...*`). If the guideline has a `## Directives` section, place the block inside it.

### 5. Implementation notes for Claude (the executing agent)

You are reading these instructions and MUST execute them literally:

- **Never write to `manual_directives:` or `suppressed_directives:`**. These lists belong to the user.
- **Never filter `directives:` based on `suppressed_directives:`**. `gov:compile`'s job.
- **Never skip the hash computation**.
- **Never silently auto-resolve hand-edits**. Interview or fail loud.
- **Always compute hashes deterministically** per Section 3a.
- **Always update both hashes** when writing a new block.
- **Always preserve `compiler_version:` drift behavior**: warn, don't auto-regenerate.
- **Always skip soft-language bullets** from `## Rules` with a warning. Never silently promote them to directives.

---

REMEMBER: Guidelines compile into the `## Team Guidelines` section of the governance index. The `## Rules` section is the source of truth — preserve MUST/NEVER language exactly. Bullets using soft language are not directives and must not be compiled. This command is the ONLY writer of the `directives:` list. `manual_directives:` and `suppressed_directives:` belong to the user. Contract locked in ADR-008.
